發表於 2020-03-24 更新於 2020-03-25

這是在建立這個blog的時候遇到的問題，在這邊記錄下來，怕之後忘記
使用環境: window 10

環境架設

  • 安裝node.js(https://nodejs.org/en/)
    • 建議安裝LTS版本，比較穩定
  • 安裝git(https://git-scm.com/)
    • 建議安裝時全部都照預設值安裝，比較不會有問題
    • 在你希望的資料夾中執行以下命令(在你想要存放hexo的資料夾按下shift並按下右鍵，選取cmd或是powershell)
      ps: 以下的blog是資料夾名稱，可以改成任何你想要的名字 這時進入localhost:4000，就可以看到你的blog了

部屬到github

  • 申請一個github帳號
  • 新建一個跟你使用者一樣名字，後面帶有.gitbub.io的Repositories
    • 如果你的名字是kevinyay945，那新建Repositories的時候就將名字設定成kevinyay945.github.io
    • 並把他設定成public
  • 修改hexo的_config.yaml，並修改以下配置(使用ctrl+F)
  • 安裝hexo部屬插件
  • 安裝完後執行以下指令即可部屬完成(若出現問題可以查看下方問題解法)
    • hexo clean 清理缓存
    • hexo generate/hexo g 本地生成
    • hexo deploy/hexo d 部署到远程仓库
    • 或者使用hexo d -g直接部署
  • 若部屬成功，可以到kevinyay945.github.io查看你的網頁了(github的page更新最慢會到30分鐘，如果點下去是404error，等個幾分鐘再重新看看)

將hexo-toc移除

再將md中撰寫code的欄位格式從```改成swig的語法

因為next有載入動畫，會影響到文章的輸出，將它關掉即可
